Concepto de Hilo


Concepto de hilo (theread) llamado también proceso ligero o subproceso, es la unidad de ejecución de un proceso y está asociado con una secuencia de instrucciones un conjunto de registros y una pila.

Cuando se crea un proceso el S.O crea su primer hilo (hilo primario) en la cual puede, a su vez, crear hilos adicionales. esto pone de manifiesto que un proceso no ejecuta, sino que es sólo el espacio de direcciones donde reside el código que es ejecutado mediante uno o más hilos. En un S.O tradicional. Cada proceso tiene un espacio de direcciones y un único hilo de control, por ejemplo, considere que un programa que incluya la siguiente secuencia de operaciones para actualizar el saldo de una cuenta bancaria cuando se efectúa un nuevo ingreso:

Saldo=Cuenta.ObtenerSaldo ();

Saldo+=ingreso;

Cuenta.EstablecerSaldo (saldo);

Cada hilo se ejecuta de forma estrictamente secuencial y tiene su propia pila en estados de los registros de UCP y su propio contador de un programa en cambio comparte el mismo espacio de direcciones, lo que significa compartir también las mismas variables globales, el mismo conjunto de ficheros abiertos, procesos hijos (no hilos hijos), señales, semáforos, etc.
