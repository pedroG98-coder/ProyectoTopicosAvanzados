Tipos de Eventos.


Un evento en C# es el modo que tiene una clase de proporcionar notificaciones a los clientes de la clase cuando ocurre algo digno de reseñar en un objeto. El uso más habitual para los eventos se produce en las interfaces gráficas; normalmente, las clases que representan controles de la interfaz disponen de eventos que se notifican cuando el usuario hace algo con el control (por ejemplo, hacer clic en un botón).

Eventos Principales

Activated: Se produce cuando el formulario se activa por código de programa o por parte del usuario. Para activar un formulario en tiempo de ejecución usando código, llame al método Activate. Este evento se puede usar para tareas tales como la actualización del contenido del formulario tomando como base los cambios efectuados en sus datos cuando no estaba activado.

Click: El evento Click pasa un EventArgs a su controlador de eventos, por lo que sólo indica que se ha producido un clic. Si necesita información del mouse más específica (botón, número de clics, rotación de la rueda o ubicación), utilice el evento MouseClick. Sin embargo, no se producirá el evento MouseClick si el clic se produce por una acción distinta de la del mouse, como presionar la tecla ENTRAR.

MouseMove: Este evento indica que cuando pase el puntero del mouse por el formulario se ejecutará lo que este codificado en él.

FormClosing: Este evento se ejecutará cuando se cierre la ventana. Por ejemplo, si queremos visualizar un mensaje de agradecimiento cuando se cierre la ventana.

ContexMenuChanged: Este evento se provoca si la propiedad ContextMenu se modifica mediante programación o interacción del usuario.

CursorChanged: Este evento se provoca si la propiedad Cursor se modifica mediante programación o interacción del usuario.

DoubleClick: Se produce cuando se hace doble clic en el control, un doble clic viene determinado por la configuración del mouse en el sistema operativo del usuario. El usuario puede establecer el tiempo entre los clics de un botón del mouse que debe considerarse como un doble clic y no como dos clics.

Load: Se produce antes de que se muestre un formulario por primera vez, este evento se puede usar para realizar tareas tales como la asignación de los recursos que utiliza el formulario.

MenúStart: Este evento se provoca cuando el usuario hace clic en cualquier elemento del menú. Este evento se puede usar para realizar tareas tales como la habilitación y deshabilitación de los controles del formulario a los que el usuario no debe tener acceso cuando se está teniendo acceso a los menús.

MouseDoubleClick: El evento MouseDoubleClick se produce cuando el usuario presiona dos veces un botón del mouse en rápida sucesión cuando el cursor está sobre el control. La configuración del mouse del sistema operativo del usuario determina el intervalo de tiempo que separa dos clics únicos de un doble clic.

Shown: El evento Shown sólo se produce la primera vez que se muestra un formulario; este evento no se genera al minimizarlo, maximizarlo, restaurarlo, ocultarlo, mostrarlo, o invalidarlo y dibujarlo de nuevo. Para obtener más información sobre el orden de eventos de un formulario, vea Orden de eventos en los formularios Windows Forms.

Layout: el evento Layout se produce cuando se agregan o se quitan controles secundarios, cuando los límites del control cambian o cuando se producen otros cambios que pueden afectar al diseño del control. El evento de diseño puede suprimirse mediante los métodos SuspendLayout y ResumeLayout. La suspensión del diseño le permite realizar varias acciones en un control sin tener que realizar un diseño para cada cambio. Por ejemplo, si se cambia el tamaño de un control y se mueve dicho control, cada operación provocará un evento Layout.

Scroll: Este evento se provoca cuando se hace clic en cualquier elemento de un menú que hace que se ejecute un comando y el menú pierda el foco. Este evento se puede usar para realizar tareas tales como la actualización del texto de un control StatusBar o la habilitación y deshabilitación de los botones de una ToolBar.
